IntelliDraft API Reference

REST API for document ingestion, project management, AI field derivation, and document generation.

Base URL: http://localhost:7071/api
🗺 Frontend Page-by-Page API Guide

🏠 Home — Projects List

1
On page load — fetch all projects
GET /api/projects
2
Search / filter projects
GET /api/projects?q=meter&status=ready
3
Delete a project (trash icon)
DELETE /api/projects/{id}

➕ Create Project (Step-by-Step)

1
User uploads PPTX / DOCX / PDF / XLSX
POST /api/upload (multipart)
2
AI reads doc and fills form fields
POST /api/extract-project-data
3
User reviews fields, clicks Save
POST /api/projects
Save the returned project_id — used in every step after this.

📋 Project Details — Tab 1 (Ingested)

1
On page open — load all fields
GET /api/projects/{id}/data
2
Render response.ingested fields in Tab 1
15 fields: name, code, unit, dates, cost, stakeholders…
3
User edits and clicks "Save Changes"
PUT /api/projects/{id}/data/ingested

🤖 Project Details — Tab 2 (Derived)

1
On page open — same call as Tab 1
GET /api/projects/{id}/data
2
Render response.derived fields in Tab 2
12 fields: challenges, requirements, workflow…
3
User clicks "Run AI Derive" (~30-60s)
POST /api/projects/{id}/derive-fields
4
User edits and clicks "Save Changes"
PUT /api/projects/{id}/data/derived

⚡ Generate Document

1
User clicks "Generate Document"
POST /api/generate/project/{id}
2
Save job_id, show sections list with spinners
Each section: pending → running → done
3
Poll every 3s until all sections are done
GET /api/generate/{job_id}
4
User clicks Download
GET /api/generate/{job_id}/export?format=docx

✏️ Review & Regenerate Sections

1
User opens a section to review
GET /api/generate/{job_id}/section/{sec_id}
2
User types feedback and submits
POST /api/…/section/{sec_id}/comment
3
Regenerate with that feedback
POST /api/…/section/{sec_id}/regenerate
4
User approves a version
POST /api/…/section/{sec_id}/accept

📐 Templates Dropdown

1
Populate template dropdown on create form
GET /api/templates?document_type=BRD
2
Pass selected template_id when saving project
POST /api/projects → { template_id }
Template controls which sections get generated (Executive Summary, Scope, etc.)

POST /upload Upload a document for parsing ▼

Uploads and parses a document. Returns a document_id used in all subsequent calls. Supported formats: .pptx, .docx, .pdf, .xlsx. Max size: 50 MB.

Request — multipart/form-data
FieldType
fileFilerequired
Example — fetch
const form = new FormData();
form.append('file', fileInput.files[0]);

const res = await fetch('http://localhost:7071/api/upload', {
  method: 'POST',
  body: form
});
const { document_id } = await res.json();
Response 201

Poll this endpoint every 3-5 seconds to track generation progress. Each section has its own status. Job is complete when all sections are done.

StatusMeaning

runningLLM currently generating
