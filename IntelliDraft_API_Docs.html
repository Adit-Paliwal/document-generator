Typical API Call Sequence

1
Upload document(s)POST /api/upload → save returned document_id
2
Auto-populate form (optional)POST /api/extract-project-data with document_ids → fill form fields
3
Save project (Step 1+2 submit)POST /api/projects → save returned project_id
4
Load Project Data sub-pageGET /api/projects/{id}/data → populate Ingested + Derived tabs
5
Start document generationPOST /api/generate/project/{id} → save returned job_id
6
Poll for progress (every 3-5 seconds)GET /api/generate/{job_id} until status === "completed"
7
Refine sections (chatbot panel)POST /api/generate/{job_id}/section/{id}/comment → POST .../regenerate
8
Export final documentGET /api/generate/{job_id}/export?format=docx

Full replace-update. Use when user explicitly edits and re-saves the complete project form.
Request Body

Same shape as POST /api/projects. All fields optional — omitted fields are left unchanged.

Response 200
PATCH
/api/projects/{project_id}
Partial autosave — only fields present in the body are written. Use for 30-second autosave while user is filling the form.
Request Body — any subset of project fields
Response 200
DELETE
/api/projects/{project_id}
Permanently delete a project and all its derived AI data. Used by Dashboard trash icon.
⚠️ This is a hard delete. No recovery possible. Show a confirmation dialog before calling.
Response

204 No Content — empty body on success.

404Project not found
GET
/api/projects/{project_id}/data
Load the Project Data sub-page. Returns both Ingested Details (user-entered) and Derived Details (AI-generated) in one call.
✅ Use this endpoint on every page load of the Project Data sub-page. Do NOT use GET /api/projects/{id} for this page.
Response 200
ℹ️ Derived fields are empty strings ("") until AI derivation is run. derived_generated_at is null until then — use this to show "AI data not yet generated" state.

Response 200

Document Generation

Project Workspace — Generate, Section Edit, Version History, Export
POST
/api/generate/project/{project_id}
Primary generation endpoint. Reads all project fields from DB — no request body needed. Use this for the project-based flow.
✅ Preferred over POST /api/generate/start. Automatically uses all saved project data.
Request Body

Empty body — all data is read from DB.

Response 201
⚠️ Project must have at least one document_id attached, otherwise returns 400.
POST
/api/generate/start
Start generation with raw document_id + user_inputs (direct flow, without a saved project).
Request Body
Response 201

Same shape as POST /api/generate/project/{id}

GET
/api/generate/{job_id}
Poll job status. Returns all sections with current Markdown content. Call every 3–5 seconds until status === "completed".
Response 200
ℹ️ status values: pendingin_progresscompletedfailed
Section-level status: pendinggeneratingcompletedfailed
GET
/api/generate/{job_id}/section/{section_id}
Get a single section with ALL versions and ALL comments. Use for the version history panel.
Response 200
POST
/api/generate/{job_id}/section/{section_id}/comment
Add a user edit request or note via the chatbot panel.
Request Body
comment_typeWhen to use

noteGeneral note, no action needed
Response 201
